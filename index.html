# TEMPLE — Auditorías Redes Dos páginas separadas, mismo repo: - **`index.html`** — pública, de solo lectura. Ranking mensual + vista por bar. Sin botones de carga ni configuración. Es lo que le compartís al resto del equipo. - **`admin.html`** — panel donde vos cargás y editás auditorías (Config, "+ Cargar auditoría", edición). No está enlazado desde `index.html` — solo entra quien tenga la URL directa. Sin backend, sin API, sin costo. La "memoria" es el propio repo: cada auditoría queda commiteada como JSON en `/data`, escrita directo desde `admin.html` vía la API de GitHub. ## Flujo (desde admin.html) 1. Puntuás cada criterio de un bar (número, comentario opcional). 2. El puntaje total y el aprobado/no aprobado se calculan solos (suma simple, umbral 80/100). No hay IA en este paso. 3. Para el plan de acción: "Copiar prompt para Claude" → pegás en una pestaña de claude.ai (gratis) → copiás la respuesta → la pegás de vuelta. 4. "Guardar auditoría" commitea todo al repo directo. ## Rúbrica - **Frecuencia (25 pts)** — mínimo 8 posteos mensuales. - **Tono y contenido (40 pts)** — según el KV que la marca envía ese mes. - **Estética visual (25 pts)** — ídem, según el KV. - **Historias destacadas (10 pts)** — Menú, Horarios y Reservas son obligatorias. - Aprobación: **80/100**. ## Puesta en marcha 1.
